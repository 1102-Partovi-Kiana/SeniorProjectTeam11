Comments

Learn proper commenting to enhance the readability of your code.

Let's Dive In

Why Comments Matter

Comments provide clarity about why certain pieces of code exist or how they function, especially for complex logic. They make code easier for other developers to grasp and your future self.

Good commenting:

  • Focuses on explaining the intent of the code rather than the obvious details.
  • Makes code easier to follow in the future.

Best Practices

1. Focus on the "Why"

# Good
# Calculate the average to monitor trends in growth.
average_height = sum(heights) / len(heights)

# Bad
# Calculate the average height.
average_height = sum(heights) / len(heights)

2. Be Clear and Concise, Avoid Over-Commenting

# Good
# Calculate total price including tax.
total_price = price * 1.09

# Bad
# This code here multiplies the price by 1.09 to account for tax and calculates the total price.
total_price = price * 1.09

Types of Comments

Single-Line Comments

Use single-line comments for brief notes.

# List Taylor Swift's top 3 albums
top_albums = ["1989", "Fearless", "Folklore"]

Block Comments

Use block comments for detailed explanations.

"""
Print numbers 1 to 100.
For multiples of 3, print "Fizz" instead of the number.
For multiples of 5, print "Buzz" instead.
For multiples of both 3 and 5, print "FizzBuzz".
"""
def fizz_buzz():
    for i in range(1, 101):
        if i % 3 == 0 and i % 5 == 0:
            print("FizzBuzz")
        elif i % 3 == 0:
            print("Fizz")
        elif i % 5 == 0:
            print("Buzz")
        else:
            print(i)

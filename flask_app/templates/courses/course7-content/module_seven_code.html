Coding the Robot Movement

Learn how to implement the logic required to use the Fetch Pick and Place Robot.

üìö Word Bank

Key terms for this module:

  • üì¶ Environment (env): The virtual space where the robot operates.
  • üîß FetchPickAndPlaceEnv: The specific environment object for the task setup.
  • üìè Distance Threshold: Maximum allowable distance between the gripper and the object.
  • üîÑ Step Size: Incremental movement of the gripper.
  • üîê Close Grip Action: Array representing the action to close the gripper.
  • üîì Open Grip Action: Array representing the action to open the gripper.
  • üí™ Lift Action: Array representing small upward movement to lift the gripper while holding an object.

Defining Constants & Actions

Before moving the robot, we define some important constants:

  • distance_threshold: Determines how close the gripper must be to the object before stopping movement.
  • close_grip_action: Instructs the gripper to close (-1 value).
  • open_grip_action: Instructs the gripper to open (+1 value).
  • lift_action: Moves the gripper slightly upwards (Z-direction).

distance_threshold = 0.01  
close_grip_action = np.array([0, 0, 0, -1])  
open_grip_action = np.array([0, 0, 0, 1])    
lift_action = np.array([0, 0, 0.1, 0])

Move the Gripper Horizontally Towards the Box

Since we don‚Äôt know exactly how many steps it will take for the gripper to reach the box, we use a while loop instead of a for loop. This loop continues running as long as the gripper is still far from the box.

What needs to happen inside the loop?

Each iteration of the loop should do the following:

  1. Compute the Horizontal Distance:
    • The gripper moves only in the X and Y directions (no vertical movement yet).
    • The horizontal distance is calculated using the Euclidean norm excluding the Z-axis.
    • We compare this distance to the threshold to decide if we should continue moving.
    Distance Calculation:
    
horizontal_distance = np.linalg.norm(box_position[:2] - gripper_position[:2])
  2. Check the Stopping Condition:
    • If the horizontal distance is less than or equal to the threshold, the gripper is close enough, and we stop moving.
    Stopping Condition:
    
if horizontal_distance <= distance_threshold:
    break  # Stop when the gripper is close enough to the box
  3. Calculate the Direction Toward the Box:
    • Find the direction vector pointing from the gripper to the box.
    • This tells us which way the gripper should move.
    Direction Calculation:
    
direction = box_position - gripper_position
  4. Normalize the Direction:
    • Since the direction vector could have any magnitude, we normalize it.
    • Normalization ensures the gripper moves at a constant speed, regardless of distance.
    Normalization:
    
direction_normalized = direction / (np.linalg.norm(direction[:2]) + 1e-8)
  5. Compute the Movement Action:
    • The gripper needs to move only in X and Y.
    • We scale the normalized direction by a step size (0.1 in this case).
    Action Calculation:
    
horizontal_action = np.array([
    direction_normalized[0] * 0.1,  # Move in X direction
    direction_normalized[1] * 0.1,  # Move in Y direction
    0,  # No vertical movement
    0   # No change in gripper state
])
  6. Take a Step in the Environment:
    • Apply the computed action to the environment so the gripper moves.
    Move the Gripper:
    
env.step(horizontal_action)

Lower the Gripper to Grasp the Box

Once horizontally aligned, the gripper needs to move downward (in the Z direction) until it is at the correct height to grasp the box. This ensures that the gripper can securely hold the object.

Now that the gripper is horizontally aligned with the box, it needs to move downward along the Z-axis until it reaches the object's surface. Since we don‚Äôt know exactly how many steps this will take, we use a while loop.

What needs to happen inside the loop?

Each iteration of the loop should do the following:

  1. Retrieve the Positions:
    • First, get the current positions of both the box and the gripper.
    • These positions are stored in variables so they can be used for calculations.
    Retrieve Positions:
    
box_position = env.sim.data.get_site_xpos('object0')
gripper_position = env.sim.data.get_site_xpos('robot0:grip')
  2. Compute the Vertical Distance:
    • Since we are only moving downward, we only compare the Z-axis values.
    • The difference between the box's height and the gripper's height gives us the vertical distance that needs to be covered.
    Distance Calculation:
    
vertical_distance = box_position[2] - gripper_position[2]
  3. Check the Stopping Condition:
    • We compare the absolute value of the vertical distance to the threshold.
    • If the distance is small enough, the gripper is close enough, and we stop moving.
    Stopping Condition:
    
if abs(vertical_distance) <= distance_threshold:
    break  # Stop when the gripper is close enough in the Z-axis
  4. Compute the Descend Action:
    • The gripper needs to move only downward (Z direction).
    • The movement step size is scale by a factor of 0.1 to ensure smooth descent.
    • No movement occurs in the X or Y directions.
    Action Calculation:
    
descend_action = np.array([0, 0, vertical_distance * 0.1, 0])
  5. Take a Step in the Environment:
    • Apply the computed descend action to move the gripper downward.
    Move the Gripper:
    
env.step(descend_action)

Example Breakdown:

Let's assume:

  • The gripper starts at Z = 0.5.
  • The box is at Z = 0.2.

# Current positions
box_position = [0.1, 0.3, 0.2]
gripper_position = [0.1, 0.3, 0.5]

# Vertical distance calculation
vertical_distance = 0.2 - 0.5  # -0.3 (negative since we move downward)

# Stopping condition check
if abs(-0.3) <= 0.01:
    break  # Stop moving

# Scale movement for smooth descent
descend_action = np.array([0, 0, -0.3 * 0.1, 0])  # [0, 0, -0.03, 0]

Close the Gripper Around the Box

After reaching the box, send a command to close the gripper. This action should securely hold the box in place.

Pause to Stabilize the Grasp

Allow the robot to hold the box for a brief moment to ensure it is stable and the grasp is firm. This avoids any unintended movements while lifting or moving the box.

Pause for 10 steps to let the gripping action stabilize, as you want to wait before moving


for _ in range(10):
env.step(np.array([0, 0, 0, 0]))

Move the Gripper and Box Towards the Ball

Calculate the direction from the box‚Äôs current position to the target ball position. Then, move the gripper (and the box it is holding) in small steps towards the ball. The movement should continue until the box is close enough to the ball, as defined by a specified distance threshold.

Now that the gripper is holding the box, it must move toward the ball. We use a for loop to ensure movement occurs in controlled steps. The loop stops when the box reaches the target location (ball) or if the maximum step count is reached.

What needs to happen inside the loop?

Each iteration of the loop should do the following:

  1. Retrieve Positions:
    • Get the current gripper position.
    • Get the current box position.
    Retrieve Positions:
    
gripper_position = env.sim.data.get_site_xpos('robot0:grip')
box_position = env.sim.data.get_site_xpos('object0')
  2. Ensure the Box Stays Gripped:
    • The gripper must continuously hold the box while moving.
    • This is done by repeatedly applying the close grip action.
    Maintain Grip:
    
env.step(close_grip_action)  # Keep gripping
  3. Measure Distances:
    • Calculate the distance from the gripper to the ball.
    • Calculate the distance from the box to the ball.
    • We check box-to-ball distance because the goal is to move the box, not just the gripper.
    Distance Calculations:
    
distance_to_ball = np.linalg.norm(ball_position - gripper_position)
distance_to_box = np.linalg.norm(ball_position - box_position)
  4. Check the Stopping Condition:
    • Once the box is close enough to the ball (below the threshold), stop the movement.
    Stopping Condition:
    
if distance_to_box <= box_to_ball_threshold:
    break  # Stop when the box is close enough
  5. Compute the Movement Direction:
    • Find the direction from the gripper to the ball.
    • Normalize this vector to ensure controlled movement.
    Normalize Direction:
    
direction_to_ball = ball_position - gripper_position
direction_normalized = direction_to_ball / (np.linalg.norm(direction_to_ball) + 1e-8)
  6. Compute the Movement Action:
    • The robot moves toward the ball by scaling the movement direction.
    • The movement occurs in all three axes (X, Y, Z) to reach the target smoothly.
    Action Calculation:
    
move_to_ball_action = np.array([
    direction_normalized[0] * 0.3,  # Move in X direction
    direction_normalized[1] * 0.3,  # Move in Y direction
    direction_normalized[2] * 0.3,  # Move in Z direction
    0  # Keep grip closed
])
  7. Move the Gripper and Box:
    • Apply two actions in the environment:
      • First, keep gripping to prevent the box from falling.
      • Then, move the gripper toward the ball.
    Move the Gripper and Box:
    
env.step(close_grip_action)  # Keep gripping
env.step(move_to_ball_action)  # Move toward the ball

Example Breakdown:

Let's assume:

  • The gripper starts at (0, 0, 0).
  • The box starts at (1, 1, 0).
  • The ball is at (3, 3, 0).
  • The threshold for stopping is 0.03.

# Current positions
gripper_position = [0, 0, 0]
box_position = [1, 1, 0]
ball_position = [3, 3, 0]

# Distance calculations
distance_to_ball = np.linalg.norm([3, 3, 0] - [0, 0, 0])  # 4.24
distance_to_box = np.linalg.norm([3, 3, 0] - [1, 1, 0])  # 2.83

# Stopping condition check
if distance_to_box <= 0.03:
    break  # Stop moving

# Normalize movement direction
direction_to_ball = [3, 3, 0] - [0, 0, 0]  # [3, 3, 0]
normalized_direction = [3/4.24, 3/4.24, 0]  # [0.707, 0.707, 0]

# Scale movement for smooth transition
move_to_ball_action = [0.707 * 0.3, 0.707 * 0.3, 0]  # [0.212, 0.212, 0]

# Apply movement
env.step(close_grip_action)  # Keep gripping
env.step(move_to_ball_action)  # Move toward the ball

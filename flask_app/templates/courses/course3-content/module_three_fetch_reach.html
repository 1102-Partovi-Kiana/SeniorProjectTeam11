Fetch Reach

Let's Dive In

Objectives

Students should be able to understand foundational programming concepts and its applications towards robotics. Students should be able to dive into a basic virtual robotics lab and make the robot move.

About the Robot

The Fetch-Reach robot is a simple robotic arm that provides an intuitive entry point into robotics programming. This robot is designed to demonstrate fundamental concepts of control and automation. The Fetch-Reach robot is equipped with several joints and actuators, allowing it to perform basic movements. The goal of this robot is to reach the destination point with the tip of the robot.

What is Given

  • global env This represents the robot's environment, which contains the robot's current state and methods for interacting with it. The environment has various “getters” and “setters”
  • env = ReachEnv() The environment object that is initialized and is part of the task setup.
  • distance_threshold The distance value that will be used to determine when the gripper is "close enough" to the ball.
  • step_size The size of each movement step that the gripper will make towards the ball.

What Needs to be Retrieved

  • ball_position This value represents the current position of the ball in the environment. The student will retrieve this by calling a method or function from the global variable “env”.
  • gripper_position() This value represents the current position of the robot's gripper. It’s essential for calculating the direction and updating the gripper’s position after each movement. It will be retrieved by calling a method or function from the global variable “env”.
  • direction The direction from the gripper to the ball, calculated by subtracting the gripper's position from the ball's position. This will be used to calculate how the gripper should move to reach the ball.

What Needs to be Coded

  • Understanding the Environment: You are provided with a robot environment (env), which holds the current state of the robot and provides functions to interact with it. Specifically, it has "getters" to retrieve information like the position of the ball and the gripper.
  • Retrieve Key Positions:  Begin by getting the current positions of both the ball and the gripper. These positions will be used to calculate the direction in which the gripper needs to move. You'll need to use the environment to obtain these positions.
  • Calculate the Direction:  The robot’s gripper must move towards the ball. To determine this, calculate the vector from the gripper’s current position to the ball’s position. This direction vector will guide the movement of the gripper.
  • Distance threshold: This value determines how close the gripper must be to the ball before it stops. If the distance between the gripper and the ball is smaller than this threshold, the gripper is considered to have reached the ball.
  • Step size:  This defines how much the gripper will move in each step. A small value is generally used to ensure smooth movement.
  • End Condition:  The loop will terminate when the distance between the gripper and the ball becomes small enough (less than the distance threshold). At that point, print the final position of the gripper to confirm it has reached near the ball.
  • Test the Outcome: Once the loop finishes, the robot's gripper should be near the ball. Print the gripper’s final position and confirm that the robot has successfully moved the gripper to the ball.
